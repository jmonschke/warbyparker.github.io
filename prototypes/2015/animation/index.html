Core animation principles

Animation on the site should:

  1. 1. Aid usability
  2. 2. Enhance the product
  3. 3. Support the brand's personality
  4. 4. Use natural motion, and
  5. 5. Almost be invisible
scroll down

Aid usability

The core focus for using motion and animation on the site is to enhance usability by guiding users through tasks.

Respond to user interaction

A well designed system provides affordances and reassuring feedback to its users. We'll always provide visual confirmation when a user interacts with our site. Why? Because leaving a user hanging, with no confirmation that their tap did anything is likely to cause an unnecessary break in their journey.

Orient the user with transitions

Abrubt changes in an interface are hard for users to process. Use animation to convey context and the relationships between objects when content transitions onto or off the screen.

Enhance the product

When used sparingly, animation can strengthen a product's story. We can use it to draw attention and highlight a new product feature. Or, we can bring a page to life through a simple head turn.

scroll down

Support the brand's personality

Warby Parker is the interesting, witty person at the party that everyone wants to know. Animations on the site should try to convey this aura and intelligence. We'll do this via smooth, natural, snappy interactions which guide users through their tasks, and seek to create a sense of wonder by occassionally delighting people on special occasions.

Use natural motion

Respect the laws of physics. Think about the mass and weight of an object and the forces which accelerate and decelerate it. Also be mindful of where the force is applied in relation to the center of mass of an object as that will determine how much it rotates.

Note that in nature, nothing moves at linear speed. In contrast, robots and machines do. To avoid feeling mechanical, we'll avoid linear timing functions and try to move objects in natural arcs.

Almost be invisible

Good animation is almost invisible. In most cases, a user should barely notice the animation as they go about their tasks. The only exception to this is when we attempt to focus their attention on something important. For example, warning a user about an error that they've just made.

Developer Guidelines

A few basic pointers when implementing animation on the site.

CSS vs javascript animation

Use CSS when you have smaller, self-contained states for UI elements. CSS transitions and animations are ideal for bringing a navigation menu in from the side, or showing a tooltip. You may end up using JavaScript to control the states, but the animations themselves will be in your CSS.

Use JavaScript when you need significant control over your animations.Something that dynamically tracks a touch position, or an animation that you need to stop, pause, slow-down or reverse typically require you to use JavaScript.
